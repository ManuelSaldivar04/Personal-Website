StickMan Math

Educational Math Game to Help Children Learn Basic Arithmetic

Description

StickMan Math was created by me and a few friends of mine at University for a semester-long project class. The goal of the project was to create an educational/helpful game or tool for the public to use. We decided to create a Pok√©mon/Mario style adventure game to teach children in a fun and educational manner the 4 basic arithmetic skills.

Features

The game is designed in an advance-by-progress style, ensuring players master the previous levels before reaching the next. Each "world" targets one of the 4 principle arithmetic operations. Players can purchase items in the shop with gold earned by defeating bosses. Items help players advance through levels, rewarding good performance.

To defeat each level's boss, players must answer math problems correctly within a time limit to deal damage to the enemy. Incorrect answers reduce the player's health. If the player loses all health before the enemy, they lose the match and cannot advance. Upon defeating an enemy, players receive gold and advance to the next level.

Book Reading Organizer

App to Organize Books Read and Books Wishing to Read

Description

Developed a C++ book organizer application at Western University to master Object-Oriented Design & Analysis (OODA) principles. Prioritized architectual scalability and maintainability over UI, implementing core functionality for managing books, reviews, and ratings within a rigorously modeled domain. Applied OODA methodology end-to-end from UML diagrams to implementation, to ensure a clean, entensible foundation.

Features & OODA Implementation

• Domain Modeling via UML Class Diagrams: Established object relationships including Book-Review aggregation (1-to-many composition) and User-List associations with strict encapsulation

• Workflow Validation with Sequence Diagrams: Modeled critical operations like "Move Book Between Lists" and "Add Review" to ensure SOLID-compliant interactions

• Use Case-Driven Development: Formalized 10+ scenarios including priority-based sorting and wish list management with preconditions/exception handling

Design Patterns

• Factory Method Pattern: Implemented to handle the creation of different types of reviews (book review, author review..). This ensures that the system can easily accommodate additional review types in the future without modifying the core logic.

  • Encourages adherence to the Open Closed principle
  • Makes the code more maintainable and readable, as review creation logic is centralized in one place

• Strategy Pattern: Implemented to allow dynamic selection of sorting algorithms (by title or author). This makes the sorting logic extensible and modular. Instead of hardcoding sorting logic, the pattern allows swapping algorithms at runtime depending on user preferences or system requirements.

  • Encourages modularity by encapsulating each sorting logic into seperate, reusable classes
  • Single-Responsibility Principle
